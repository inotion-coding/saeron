# DESIGN.md — 새론 학원 홈페이지 디자인 시스템

> 색상-무관(color-agnostic) 디자인 명세. **색상 값은 추후 확정** — 의미(semantic)와 토큰 이름만 먼저 고정하고, 확정 시 §2 토큰 값만 교체한다.
> 개발 절차는 [PROCESS.md](PROCESS.md), 작업 규칙은 [CLAUDE.md](CLAUDE.md) 참조.

---

## 1. 디자인 원칙

입시/종합 학원 사이트의 핵심은 **신뢰감**과 **정보 전달력**이다.

1. **신뢰감 우선** — 정돈된 그리드, 일관된 간격, 과하지 않은 장식
2. **가독성 우선** — 충분한 본문 크기/행간, 명확한 정보 위계(제목→소제목→본문)
3. **콘텐츠 우선** — 디자인이 정보를 가리지 않게. 여백으로 호흡 확보
4. **상담 전환 동선** — 주요 지점마다 상담 CTA가 자연스럽게 노출
5. **모바일 우선** — 학부모/학생 다수가 모바일 접근 → 모바일에서 먼저 완성

## 2. 디자인 토큰

**원칙: 컴포넌트는 토큰만 참조한다. hex 직접 사용 금지.**
`styles/globals.css`의 CSS 변수로 정의하고 `tailwind.config`의 theme에서 매핑한다.

### 2.2 타이포그래피

- **폰트 패밀리**: 본문/제목 모두 `Pretendard`(후보), 폴백 `system-ui, sans-serif`
- **유동 스케일(권장)**: 제목급은 `clamp(최소, 뷰포트비례, 최대)`로 정의해 폭에 따라 **연속적으로** 커지게 한다. 예: `--fs-display: clamp(2rem, 1.2rem + 4vw, 3.5rem)`. 본문은 가독성 위해 1rem 고정 또는 좁은 clamp.
- **타입 스케일** (아래는 최소~최대 범위. 단계적 점프가 아니라 그 사이를 유동으로 채움)

| 토큰 | 용도 | 크기(rem) | 굵기 | 행간 |
|------|------|-----------|------|------|
| `display` | Hero 대제목 | 2.5–3.5 | 700 | 1.15 |
| `h1` | 페이지 제목 | 2.0 | 700 | 1.2 |
| `h2` | 섹션 제목 | 1.5 | 600 | 1.3 |
| `h3` | 카드/소제목 | 1.25 | 600 | 1.4 |
| `body` | 본문 | 1.0 | 400 | 1.6 |
| `small` | 보조/캡션 | 0.875 | 400 | 1.5 |

### 2.3 간격 (spacing scale)

4px 기반 스케일: `4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96` (px). Tailwind 기본 스케일과 호환.

### 2.4 라운드 / 그림자 / 브레이크포인트

```css
--radius-sm: 6px;   --radius-md: 10px;  --radius-lg: 16px;  --radius-full: 9999px;
--shadow-sm: 0 1px 2px rgba(0,0,0,.06);
--shadow-md: 0 4px 12px rgba(0,0,0,.08);
--shadow-lg: 0 12px 32px rgba(0,0,0,.12);
```

| 브레이크포인트 | 폭 | 대상 |
|----------------|-----|------|
| (base) | < 640px | 모바일 |
| `sm` | ≥ 640px | 큰 모바일 |
| `md` | ≥ 768px | 태블릿 |
| `lg` | ≥ 1024px | 데스크톱 |
| `xl` | ≥ 1280px | 와이드 |

## 3. 레이아웃 시스템

- **컨테이너 최대폭**: `lg` 기준 1120–1200px, 좌우 패딩 모바일 16px / 데스크톱 24px
- **그리드**: 12컬럼 기반 사고. 카드 그리드는 모바일 1열 → `md` 2열 → `lg` 3열
- **섹션 수직 리듬**: 섹션 간 상하 패딩 모바일 48–64px / 데스크톱 96px로 일관
- **정렬**: 본문 텍스트 좌측 정렬, Hero/섹션 헤더는 상황에 따라 중앙

## 4. 컴포넌트 인벤토리

각 컴포넌트는 변형(variant)과 상태(state: default/hover/focus/disabled)를 정의한다.

| 컴포넌트 | 변형 | 비고 |
|----------|------|------|
| `Button` | primary / secondary / ghost | hover·focus·disabled 필수. focus 링 표시 |
| `Container` | — | 최대폭+좌우패딩 래퍼 |
| `Section` | default / muted(배경 surface) | 수직 리듬 통일 |
| `Header / Nav` | 데스크톱 가로 / 모바일 햄버거 | 현재 페이지 active 표시, sticky 검토 |
| `Footer` | — | 연락처·주소·운영시간·간단 메뉴 |
| `Hero` | 홈용 대형 / 서브페이지용 소형 | 제목+부제+CTA |
| `Card` | 기본 | 강사·프로그램 카드의 베이스 |
| `TeacherCard` | — | 사진·이름·담당과목·한줄 이력 |
| `ProgramCard` | — | 과정명·대상·내용·특징 태그 |
| `ScheduleTable` | 데스크톱 테이블 / 모바일 카드형 | §7 반응형 참조 |
| `ContactForm` | — | 이름·연락처·문의내용, 검증·에러 상태 |
| `CTASection` | — | "상담 신청" 유도 배너 |
| `Badge / Tag` | — | 과목·대상 라벨 |

## 5. 페이지별 와이어프레임

### 홈 (랜딩)
```
[ Header / Nav ]
[ Hero: 슬로건 + 부제 + (상담 신청 CTA) ]
[ 학원 강점 3~4개 요약 (아이콘 카드) ]
[ 프로그램 미리보기 (카드 일부 + 전체보기 링크) ]
[ 강사 미리보기 (대표 강사 카드) ]
[ 합격/성과 또는 후기 (선택) ]
[ CTASection: 상담 신청 ]
[ Footer ]
```

### 학원 소개 (/about)
```
[ 서브 Hero: "학원 소개" ]
[ 교육 철학 / 미션 ]
[ 학원 강점 상세 ]
[ 연혁 · 시설 (있으면) ]
[ CTASection ]
```

### 강사 소개 (/teachers)
```
[ 서브 Hero ]
[ TeacherCard 그리드 (1→2→3열) ]
```

### 프로그램 (/programs)
```
[ 서브 Hero ]
[ 대상/학년별 섹션 ]
[ ProgramCard 그리드 ]
[ CTASection ]
```

### 시간표 (/schedule)
```
[ 서브 Hero ]
[ ScheduleTable (요일 × 시간) ]
[ 안내 문구 (변동 가능 등) ]
```

### 상담 문의 (/contact)
```
[ 서브 Hero ]
[ 2열: ContactForm  |  연락처·운영시간·오시는 길(지도) ]   (모바일 1열)
```

## 6. 접근성 (a11y)

- **색 대비**: 본문 텍스트 대비 WCAG AA(4.5:1) 이상 — 색상 확정 시 반드시 검증
- **포커스**: 키보드 포커스 시 명확한 focus 링(절대 `outline: none`만 두지 않음)
- **시맨틱 마크업**: `header/nav/main/section/footer`, 제목 레벨 순서 준수
- **이미지**: 의미 있는 이미지에 `alt`, 장식 이미지엔 빈 `alt`
- **폼**: 모든 입력에 `label` 연결, 에러 메시지 텍스트 + `aria-invalid`
- **터치 타깃**: 최소 44×44px

## 7. 반응형 동작

> **원칙: 유동 우선(fluid-first).** 이용자는 휴대폰·태블릿·노트북 등 매우 다양한 폭으로 접속하므로, 브레이크포인트 사이의 임의 폭(예: 860px)에서도 픽셀 단위로 깔끔하게 맞아야 한다. 특정 폭에서만 정상이고 그 사이가 어긋나면 불합격.

### 유동 구현 가이드
- **고정 px 폭 지양** — `%`, `rem`, `min()/max()/clamp()`, grid `fr`/`repeat(auto-fit, minmax())`, flex로 연속 변형
- **브레이크포인트는 구조 전환에만** — 1열↔다열, 가로메뉴↔햄버거 같은 큰 전환에 한정. 미세 조정은 유동 단위가 흡수
- **카드 그리드 권장 패턴**: `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))` — 폭에 따라 열 수가 자동 증감(고정 1/2/3열 점프 대신)
- **넘침 방지**: flex/grid 자식에 `min-width:0`, 긴 텍스트 `break-words`, 이미지 `max-width:100%`, 가로 스크롤 0
- **컨테이너 기준 적응**: 카드·시간표 등은 페이지가 아닌 부모 폭에 반응 → container query 검토
- **검증**: DevTools에서 320px→1920px 폭을 **연속적으로 드래그**하며 깨짐·겹침·가로 스크롤 없는지 확인 (PROCESS §3-1)

### 컴포넌트별 동작
- **내비게이션**: 데스크톱 가로 메뉴 → 좁아지면 햄버거 + 슬라이드/드롭 패널
- **카드 그리드**: `auto-fit/minmax`로 폭에 따라 1→2→3열 자동
- **시간표**: 넓은 폭은 가로 테이블, 좁아지면 가로 스크롤(`overflow-x`) 또는 요일별 카드/아코디언으로 전환
- **2열 레이아웃**(상담 폼/소개): 넓은 폭 2열 → 좁아지면 1열 세로 적층
- **이미지**: `next/image`로 반응형 크기·지연 로딩, `sizes` 지정
